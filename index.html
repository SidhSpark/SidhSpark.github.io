0.2 The Explorer's Compass: Windows Command Prompt 0.2 ගවේෂකයාගේ මාලිමාව: Windows Command Prompt

This is how you talk to your computer directly with text commands. Don't worry, it's simpler than it looks! To open it: Press the Windows Key, type cmd, and press Enter. මේ, ඔබේ පරිගණකය සමඟ text විධාන මගින් කෙලින්ම කතා කරන ක්‍රමයයි. බය වෙන්න එපා, හිතනවට වඩා සරලයි! විවෘත කිරීමට: Windows Key එක ඔබා, cmd ලෙස type කර, Enter ඔබන්න.

Basic "Spells" (Commands): මූලික "මන්ත්‍ර" (විධාන):

  • dir: Shows files and folders in your current location (like looking inside a folder in File Explorer). ඔබ දැන් සිටින තැන (ෆෝල්ඩරයේ) ඇති files සහ folders පෙන්වයි (File Explorer එකේ ෆෝල්ඩරයක් ඇතුල බැලීම වගේ).
  • cd : Changes directory (like double-clicking a folder to open it). Example: cd Downloads වෙනත් ෆෝල්ඩරයකට මාරු වීම (ෆෝල්ඩරයක් double-click කර open කිරීම වගේ). උදා: cd Downloads
  • cd ..: Goes up one folder level (like clicking the "Up" arrow in File Explorer). එක් ෆෝල්ඩර මට්ටමක් උඩට යාම (File Explorer එකේ "Up" arrow එක click කිරීම වගේ).
  • md : Makes a new directory/folder (like Right-click > New > Folder). Example: md MyNewProject අලුත් ෆෝල්ඩරයක් සෑදීම (Right-click > New > Folder වගේ). උදා: md MyNewProject

Try navigating to your Desktop and creating a folder named "TestFolder". You're getting the hang of it! ඔබේ Desktop එකට ගොස් "TestFolder" නමින් ෆෝල්ඩරයක් සාදා බලන්න. ඔබට දැන් මේක තේරෙනවා!

Look! A package.json file has appeared. That's your project's official ID. You've just laid the foundation for a professional project! දැන් බලන්න. package.json නමින් file එකක් හැදිලා! එය තමයි අපේ project එකේ හැඳුනුම්පත. ඔබ දැන් project එකකට පදනම දැම්මා!

3.1 The Time Machine & Cloud Safe-House: Git & GitHub 3.1 කාල යන්ත්‍රය සහ ඔන්ලයින් සුරක්ෂිතාගාරය: Git & GitHub

You've installed Git and created a GitHub account. Now, let's use them for your project. ඔබ Git install කර GitHub ගිණුමක් සෑදුවා. දැන්, ඒවා ඔබේ ව්‍යාපෘතිය සඳහා භාවිත කරමු.

Let's create our first "Save Point" in your project folder using the VS Code terminal. VS Code terminal එක භාවිත කර, ඔබගේ project ෆෝල්ඩරයේ පළමු "Save Point" එක නිර්මාණය කරමු.

3.2 Sharing with the World: Pushing to GitHub 3.2 ලෝකය සමඟ බෙදාගැනීම: Pushing to GitHub

Now, let's send your local creation to your secure online safe-house (your GitHub account). දැන් ඔබගේ දේශීය (local) නිර්මාණය, ලෝකයට පෙනෙන ඔන්ලයින් සුරක්ෂිතාගාරයට (ඔබගේ GitHub ගිණුමට) යවමු.

  1. On GitHub, click the "+" icon in the top right and select "New repository". GitHub හි, ඉහළ දකුණේ ඇති "+" අයිකනය click කර "New repository" තෝරන්න.
  2. Name your repository my-digital-footprint. Make sure it's "Public". Click "Create repository". ඔබගේ repository එක my-digital-footprint ලෙස නම් කරන්න. එය "Public" බවට වග බලා ගන්න. "Create repository" click කරන්න.
  3. GitHub will show you commands for an "existing repository". Copy these lines (it will look similar to below, but with *your* username): GitHub "existing repository" එකක් සඳහා විධාන පෙන්වයි. එම පේළි copy කරගන්න (එය පහත ආකාරයට සමාන වනු ඇත, නමුත් *ඔබගේ* username එක සමඟ): 
git remote add origin https://github.com/YOUR_USERNAME/my-digital-footprint.git
git branch -M main
git push -u origin main

Paste and run these commands one by one in your VS Code terminal. Refresh your GitHub page. Your files are now stored safely online! This is huge! Your code is now global and secure! මෙම විධාන එකින් එක copy කර VS Code terminal එකේ run කරන්න. ඔබගේ GitHub පිටුව refresh කරන්න. ඔබගේ files දැන් සුරක්ෂිතව online තැන්පත් වෙලා! මෙය විශිෂ්ටයි! ඔබගේ නිර්මාණ දැන් ලෝක ව්‍යාප්තයි!

Your project is on GitHub, but it doesn't have a live web address. Let's fix that in 60 seconds using a service called Vercel. අපේ ව්‍යාපෘතිය GitHub එකේ තිබුනට, එයට live web address එකක් නැහැ. Vercel වැනි සේවාවක් භාවිත කර, මිනිත්තුවකින් එය සජීවී කරමු.

  1. Go to Vercel.com and sign up with your GitHub account. It's free. Vercel.com වෙත ගොස්, ඔබගේ GitHub ගිණුමෙන්ම ලියාපදිංචි වන්න.
  2. Choose to "Add New..." > "Project", and select your `my-digital-footprint` repository (you may need to authorize Vercel to access it). "Add New..." > "Project" තෝරා, ඔබ දැන් සෑදූ `my-digital-footprint` repository එක තෝරන්න (Vercel වෙත එයට ප්‍රවේශ වීමට authorize කිරීමට සිදුවිය හැක).
  3. Vercel will automatically detect it's a static site. Don't change any settings. Just click the "Deploy" button. එය static site එකක් බව Vercel ස්වයංක්‍රීයව හඳුනාගනීවි. කිසිදු setting එකක් වෙනස් නොකර, "Deploy" button එක ඔබන්න.
